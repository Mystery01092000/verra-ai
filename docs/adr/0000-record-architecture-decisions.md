# ADR-0000: Record architecture decisions
**Status:** Accepted · 2026-06-16
## Context
We need a durable, reviewable history of significant technical decisions for Verra.
## Decision
Use lightweight ADRs (Michael Nygard format) in `docs/adr/`, numbered and immutable. Each has
Status, Context, Decision, Consequences. Supersede with a new ADR instead of editing.
## Consequences
+ Shared understanding and onboarding; clear rationale.
− Small authoring overhead per decision.
